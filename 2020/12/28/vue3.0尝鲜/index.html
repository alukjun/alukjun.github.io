vue3.0尝鲜

Posted on 2020-12-28

前序

随着前端技术的日益更新，vue3.0框架应运而生，正所谓学无止境，在前端这条路上没有尽头，让我们走进vue3.0

vue2.x -> vue3.0 变化

vue3.0可以完全支持vue2.0，所以说对于老项目的升级来说不会那么的困难，升级以后不需要更改关于Vue的代码,但是你引用的插件就…,看运气了.

vue3.0创建项目

1、vite

优点：速度快（不需要通过webpack编译）

缺点： 不稳定（目前来说），需要自己安装vue-router.,vuex, (敲重点,vue-router, vuex都要用4.0, 安装的时候容易出错(出错多安装几次就行了)

2、vue-cli

优点：插件齐全无需自己安装

缺点：依赖较多安装较慢

变化一 main.js

main.js里面变了 多了createApp方法

由于这里用了 import { createApp } from ‘vue’; 而不是import Vue from ‘vue’, 所有使用一些Ui框架,以及第三方的包都会报以下的错.

遇到这个报错,基本上就说明你用的这个插件不支持Vue3.0了.

变化二 template模板部分

vue2.0里 template模板里只能有一个根标签, 而3.0可以使用多个根标签

变化三 Composition API（组合api）

在vue2.0里我们都把.vue文件里的js部分叫做Function API, 而在3.0里变为Composition API 这里是变化最大的地方，看下图

(敲重点: import { setup, watch, reactive, toRefs, onMounted } from “vue”)

Composition API 是什么？

setup

setup 函数 是一个新的Vue组件， 是Composition API 入口

数据声明

vue2.0 里数据都是在data()里声明， 而在3.0里也取消了data(), 引用了Reactive(), ref()

ref() 声明单一基础数据类型时使用，Reactive()声明单一对象时使用

还有其他改变生命周期
